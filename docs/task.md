# 项目 自平衡瞄准小车

## 任务

设计一个两轮自平衡小车，小车能够沿一条圆形黑线（圆环）稳定行进。在圆环的中心位置竖直固定一个矿泉水瓶，瓶身贴有白色UV感光纸张。小车车身安装一个激光发射器（激光功率符合安全标准），要求小车在绕圆环行进过程中，激光点始终照射在感光纸张上，且感光纸上激光点的上下波动幅度尽可能小。

## 要求

1. 基本要求
  1. 将小车放置在起跑线，一键启动后能够立刻实现稳定站立 3 秒，前后移动幅度不超过10cm；
  2. 能够直立沿圆环黑线稳定行进完整一周（360°），行进过程中不得脱离黑线超过 2 秒；
  3. 完成一周后，小车回到出发点附近，稳定站立并发出声、光提示；
  4. 在整个行进过程中，小车上的激光点必须始终照射在矿泉水瓶的白色UV感光纸上（允许短暂抖动，但离开纸张时间累计不超过2 秒）。
2. 发挥部分
  1. 小车能够沿圆环连续行进两周，且激光点始终保持在感光纸范围内；
  2. 在行进过程中，感光纸上激光点的上下波动幅度（即激光点中心与感光纸几何中心的垂直距离最大值）≤2cm；
  3. 小车发车时能够实现自主原地起立（即一键启动前小车可任意倾倒，启动后不可用外力搀扶）；
  4. 在完成上述任务的前提下，行进速度越快越好（按完整一周的时间计）。

## 说明

1. 小车底盘可附加倾倒保护装置，但车轮落地时，车身不可依靠保护装置实现直立，小车可自由倾倒。倾倒时，车身垂直轴线与地面水平线夹角θ（见附录示意图）不大于60 度，要求在车身标记中轴线，方便检查；
2. 要求使用 TI（推荐 M0 系列）或瑞萨的主控芯片作为小车的核心控制器件。若使用带有其他微控制器的摄像头模块应只用来进行图像识别与分类功能，摄像头模块拆除后仍可实现直立。
3. 圆环黑线内径约80cm，黑线宽度约 1.8cm。圆环中心放置一个普通矿泉水瓶（高度约 23cm），瓶身用白色UV 感光纸包裹（纸张包裹瓶身后，整体高度不可超过25cm）。激光点应照射在纸张范围内，纸张上可观察光斑位置。激光发射器波长405nm，功率≤5mW（安全等级Class3R以内）。激光器由小车独立供电，不得使用外部对准辅助。
4. 小车底盘仅包含如附图所示的电机、车轮和一块铝合金支撑板。

# 项目规划

## 系统分工

- **TI MSPM0G3507（LaunchPad）—— 平衡车主控**：负责姿态解算、电机驱动、编码器测速、平衡与基础运动控制、声光提示、与 K230 串口通讯。
- **K230（CanMV）—— 视觉与执行机构控制器**：负责双摄像头采集与处理、循迹解算、激光光斑识别、**云台双舵机 PWM 直驱**、路径与瞄准决策。
- **通讯链路**：MSPM0G3507 ↔ K230，单路 UART（带 DMA），定长帧协议，双向心跳。主控向 K230 推送 IMU pitch / 车速 / 状态，K230 向主控回传期望前进速度 v 与角速度 ω 等运动指令。
- **责任边界**：TI 主控只对"车体本身"负责（站得住、走得动），所有"看世界"和"对靶"的工作全部下放到 K230。

## 硬件清单

1. **主控**：MSPM0G3507 LaunchPad（Cortex-M0+ 80 MHz，带 MathACL，无 FPU）。
2. **姿态传感器**：优先 MPU6050（6 轴，互补滤波 / Mahony 即满足平衡需求）；备选 ATK-MS901M（自带四元数，但磁力计易被电机 PWM 干扰，且其十五阶 EKF 不在主控运行）。
3. **电机**：双路减速直流电机，**已确认自带正交编码器**（4 引脚：A 相 / B 相 / VCC / GND，电机动力线另走 2 线）。编码器输入接 MSPM0G3507 **硬件 QEI** 通道，避免 GPIO 软解码丢脉冲。
4. **电机驱动**：TB6612FNG，PWM 15~25 kHz，避开人耳与 IMU 通带。
5. **云台**：市面成品双轴舵机云台（仅由两只舵机机械组装，**不含任何驱动板**）。**舵机 PWM 由 K230 直出**，不再占用主控资源，也不再讨论"独立云台 MCU"方案。
6. **激光发射器**：405 nm、≤5 mW（Class 3R 以内），独立电池供电；开关 GPIO 由主控或 K230 控制。
7. **声光提示**：蜂鸣器 + LaunchPad 板载 LED1 / RGB LED2（如不冲突可保留板载跳线，否则断开 J4~J7 释放 PA0 / PB22 / PB26 / PB27）。
8. **调试**：板载 XDS110 UART（PA10 / PA11）作为开发期日志；运行期遥测建议另开一路蓝牙串口以避免与下载链路冲突。

## 设计要点

1. **平衡环独占主控高频时片**：循迹与云台都搬到 K230 后，MSPM0G3507 的中断密度大幅下降，平衡环可稳定跑在 200~500 Hz。
2. **云台 IMU 前馈在 K230 实现**：主控以 ≥100 Hz 把 pitch 角随状态帧推给 K230，K230 把它反向叠加到俯仰舵机命令上做载体扰动补偿（架构同 *〔Claude 批注 C-9〕*，仅执行单位从主控变为 K230）。
3. **K230 拥有运动决策权**：循迹算法看见的横向偏差和环上相位角，由 K230 直接生成 (v, ω) 并下发给主控；主控只做执行。
4. **失效安全**：与 K230 通讯心跳超时（如 200 ms），主控自动退化到"原地稳定平衡 + 报警"模式，绝不空跑指令。

> 本次相对原计划的变更说明：
> - 经调研，市售成品双轴云台均不附带驱动板，**云台舵机 PWM 改由 K230 直接输出**；与之相关的 *〔Claude 批注 C-3 / C-9〕* 中"是否自制 / 是否独立 MCU"两小问随之关闭，但 MG90S 带宽与精度限制依旧成立，仍需在调试阶段评估是否升级数字舵机。
> - 经核对，已购电机自带 4 引脚正交编码器，**编码器从"待选"升级为"已确定"**，硬件 QEI 引脚成为引脚分配的核心约束（弥补 *〔Claude 批注 C-4 / C-5〕* 指出的关键缺漏）。

# 执行计划

## 阶段 0：硬件准备与引脚分配

1. 整理 LaunchPad 跳线现状（参考 `pin.md`）：
   - 板载 LED 占用 PA0 / PB22 / PB26 / PB27（J4~J7），按用途决定保留或断开。
   - XDS-UART0 默认接到 PA10 / PA11（J21 / J22），保留为开发日志口。
   - **J12 已将 PA29 / PA30 / PB14 引到 QEI 接口，可直接复用作为一路硬件正交解码输入**，另一路 QEI 单独安排在另一组定时器通道。
2. 输出 **引脚分配表**（信号名 / 方向 / 外设 / 备注）作为唯一真源，至少覆盖：
   - 双路 **硬件 QEI**（左 / 右编码器，A / B 相）；
   - TB6612：4×PWM 或 2×PWM + 4×方向 + 1×STBY；
   - IMU：SPI（推荐）或 I²C，含 INT；
   - K230 通讯：独立 UART + DMA，**不复用 XDS-UART**；
   - 蜂鸣器、一键启动按键、电池分压 ADC、激光器使能；
   - （可选）蓝牙串口。
3. 通电先空载验证 5V / 3V3 电源与电机驱动板、舵机供电相互独立、共地正确。

## 阶段 1：姿态传感器与通讯链路

1. 接入 MPU6050，编写驱动与互补滤波，目标：俯仰静态噪声 < 0.1°、动态滞后 < 10 ms，串口波形可视化。
2. 搭建 MSPM0G3507 ↔ K230 UART 帧通讯：
   - 帧格式：`0xAA 0x55 | LEN | CMD | PAYLOAD | CRC16 | 0x55 0xAA`；
   - 必备帧：`IMU_TELEM`（pitch、pitch_rate、车速、状态位）、`MOTION_CMD`（期望 v、ω、模式）、`HEARTBEAT`、`ERROR`；
   - 双向心跳与超时降级策略。
3. K230 端先做串口收发回环测试，确认链路稳定（误码率 / 丢包率）。

## 阶段 2：电机驱动与编码器闭环

1. 编写 TB6612 驱动，完成正反转、刹车、滑行三种模式与 PWM 频率调试。
2. 配置硬件 QEI 解码两路编码器，在示波器 / 串口验证脉冲数线性度与方向位正确，**确认无丢脉冲**。
3. 单轮速度闭环（PI）：阶跃响应稳定无超调，左右轮一致性 ≤ 5%。
4. 标定轮径、编码器单位转换系数与电机死区，写入 Flash。

## 阶段 3：平衡控制

1. 静止状态先调俯仰平衡环（PD 起步，后期可升级为 LQR）。
   - *结合轮径、质量、重心位置等先验数据建立倒立摆模型，做基于模型的参数整定，比纯试凑高效得多*（呼应 *〔Claude 批注 C-7〕*）。
2. 叠加速度环（外环 PI 输出俯仰参考角 → 内环平衡环），消除贴地慢漂。
3. 验收：原地直立 ≥ 3 s，前后偏移 ≤ 10 cm，满足基础项 1。

## 阶段 4：基础运动与运动执行接口

1. 实现 `setVelocity(v, ω)` 接口：把 K230 给的纵向速度与角速度翻译为左右轮目标转速。
2. 直行 / 转弯 / 启停轨迹质量调试，目标：直线 1 m 横向偏差 < 5 cm。
3. 把该接口作为 `MOTION_CMD` 的执行目标，由 K230 远程开环驱动，验证全链路时序与延迟。

## 阶段 5：K230 视觉、瞄准与云台控制（K230 侧主战场）

1. **激光光斑检测**（摄像头 A，朝向瓶子）：405 nm UV 激发感光纸荧光，窄带滤光 + 阈值二值化得到光斑中心；输出 (Δx, Δy) 相对感光纸中心。
2. **黑线循迹**（摄像头 B，朝向地面）：
   - 用主控推送的 pitch 角做 **IPM 逆透视变换**，消除车体俯仰造成的画面"地面跳动"（呼应 *〔Claude 批注 C-8〕*）。
   - 圆环几何先验已知（半径 ≈ 40 cm），算法输出 **圆上相位角 φ** 与 **横向偏差 e_y**，进一步生成 (v, ω)。
3. **云台双舵机控制（K230 PWM 直出）**：
   - 俯仰：**前馈** = 主控 pitch 角反向叠加；**反馈** = 光斑 Δy 的 PID 修正。
   - 偏航：以相位角 φ 算开环目标方位为主，光斑 Δx 做小幅闭环修正，避免视觉延迟引起振荡。
4. 通讯联调：闭合"K230 看 → 算 → 给主控发 v / ω + 自驱云台"完整链路；舵机零位标定流程上线。

## 阶段 6：完整循环与基础项验收

1. 起跑线一键启动 → 站立 3 s → 沿圆环行进一周 → 回到起点声光提示，满足基础项 1~3。
2. 全程激光保持在感光纸内（基础项 4），主控在 K230 通讯丢失时自动退化为原地平衡。

## 阶段 7：发挥项调优

1. **激光波动 ≤2 cm**：核心在云台前馈增益与舵机机械零位的反复整定；考虑把激光器安装得尽量靠近车体俯仰轴，减小线臂引起的位移扰动；必要时把 MG90S 升级为数字舵机以提高带宽。
2. **自主原地起立**：检测倾倒方向 → 反向梯形速度曲线甩起 → 接近垂直时切回平衡环；准备保护垫与慢动作录像辅助调参。
3. **连续两周**：电池续航与累计漂移考核；以相位角 φ 累计实现严格的圈数判定。
4. **速度优化**：最后处理。瓶颈通常是过弯离心力、视觉处理延迟与平衡环增益裕度。

## 阶段 8：稳定性与失效保护

1. 跌倒检测（pitch > 60° 立即切电机 PWM）、低电压保护、舵机过流监测、看门狗。
2. 通讯心跳超时降级与告警策略。
3. 整车上电自检 + 标定数据加载流程，所有标定参数固化在 Flash。
